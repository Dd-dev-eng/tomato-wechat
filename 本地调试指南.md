# 🍅 主动番茄 - 本地开发调试指南（超简单版）

## ✅ 好消息！已经不需要安装 MongoDB 了！

我已经把项目改成用 **内存数据库** 了——完全不用安装任何东西！

---

## 🚀 本地调试只需 3 步！

---

## 第一步：获取微信测试号配置（5分钟）

1. 访问 https://mp.weixin.qq.com/debug/cgi-bin/sandbox
2. 用微信扫码登录
3. 你会看到：
   - **AppID**（复制它）
   - **AppSecret**（复制它）
4. 找到「接口配置信息」部分：
   - **Token**：自己想一个，比如 `tomato_123`（记住它！）
5. 找到「测试号二维码」，用你的微信扫一扫关注测试号

---

## 第二步：配置 .env 文件

编辑项目根目录下的 `.env` 文件，填入你的配置：

```env
PORT=3000
USE_MEMORY_DB=true
MONGODB_URI=mongodb://localhost:27017/tomato_db
WECHAT_APPID=填入你的AppID
WECHAT_APPSECRET=填入你的AppSecret
WECHAT_TOKEN=填入你刚才设置的Token
WECHAT_ENCODING_AES_KEY=（测试号可以留空）
```

---

## 第三步：启动应用！

在项目目录下运行：

```bash
npm start
```

你应该看到：
```
🎉 内存 MongoDB 已启动（调试模式）
服务器运行在端口 3000
```

---

## 第四步：让微信找到你的应用（ngrok）

### 安装 ngrok
1. 访问 https://ngrok.com/download
2. 下载 Windows 版本
3. 解压到一个文件夹

### 启动 ngrok
打开一个新的终端，运行：
```bash
ngrok http 3000
```

启动后，你会看到类似这样的信息：
```
Forwarding  https://abc123.ngrok-free.app -> http://localhost:3000
```

**复制那个 HTTPS 开头的地址**（比如 `https://abc123.ngrok-free.app`）

---

## 第五步：配置微信公众号服务器

回到微信测试号配置页面：

1. 在「接口配置信息」中：
   - **URL**：填入 `https://你的ngrok地址/wechat`
   - **Token**：填入和 .env 里一样的 Token
2. 点击「提交」按钮

## 常见问题

### Q：ngrok 链接为什么总是变？
A：免费版 ngrok 每次重启都会换链接，测试时注意重新配置微信。

### Q：应用运行但是微信收不到回复？
A：检查微信后台服务器配置是否保存成功，检查 ngrok 是否在运行。

### Q：Token 验证失败？
A：确认 .env 里的 Token 和微信后台填的完全一致。

### Q：重启应用后数据丢失？
A：因为我们用的是内存数据库，数据只在应用运行时存在，重启就清空了。正式部署时会用真实数据库。
