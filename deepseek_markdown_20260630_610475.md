## 1. 产品概述

### 1.1 产品愿景
一个运行在微信聊天框里的**主动式番茄钟**。帮助用户改善拖延、主动把握时间，通过高频的“我接下来要做什么”的主动承诺，让时间的流逝变得可见且有掌控感。

### 1.2 核心价值
-   **主动承诺**：每次做事前，都需手动确认“活动 + 时长”，把注意力拉回当下。
-   **轻量提醒**：时间到了准时提醒；超时不操作会再次提醒。只记录，不惩罚。
-   **正向反馈**：用“完美番茄”和“半熟番茄”给予每一次主动行为正向激励。
-   **微信内闭环**：无需跳出微信，用熟悉的聊天界面管理时间和注意力。

### 1.3 目标用户
任何想改善拖延、希望对自己的时间花费有觉知，但不喜欢复杂工具的个人用户。

## 2. 产品形态与技术平台

-   **形态**：微信公众号（服务号）
-   **交互载体**：聊天框对话 + 底部自定义菜单 + 模板消息提醒
-   **用户路径**：关注即用，无需下载App

## 3. 核心概念定义

| 概念 | 定义 |
| :--- | :--- |
| **月/周方向标签** | 用户设定的大方向，如“阅读、运动、学习”，仅作为每日设定活动的快捷选用来源。 |
| **今日活动池** | 每天早晨用户设定的、今天可能做的一组具体活动，可动态增删。 |
| **当下承诺** | 每次从活动池选择或手动输入一个活动，并设定时长，然后立即开始计时。这是产品的灵魂。 |
| **完美番茄** | 在收到“时间到”提醒后点击“完成”结束的活动；或超时后，在当天主动点击“结束”的活动。 |
| **半熟番茄** | 在计划时长到达之前，由用户主动提前结束的活动。 |

## 4. 功能需求详情

### 4.1 每日晨间提醒与设定

-   **触发**：每日早上 8：00（可自定义），通过模板消息推送。
-   **消息内容**：“早上好，今天你想把时间花在哪几个活动上？点击设定今日活动池。”
-   **设定交互**：
    1.  **从标签快捷添加**：显示用户已保存的月/周标签，勾选后自动生成同名活动。
    2.  **手动输入**：直接发送文字活动名，添加到今日活动池。
-   **动态管理**：用户可随时发送“添加活动 [活动名]”或“删除活动 [活动名]”调整活动池。

### 4.2 发起活动

-   **入口**：
    -   底部菜单点击“开始活动”
    -   聊天框发送“开始”或“新活动”
-   **步骤1：选择活动名**
    -   系统回复一张卡片，包含：
        1.  **今日活动池快捷选择**：每个活动都是一个可点击按钮。
        2.  **手动输入按钮**：点击后，用户可直接发送新活动名。
    -   用户选择或输入后，系统确认：“你选择了【阅读】”。
-   **步骤2：设定时长**
    -   系统再发送时长选择卡片，包含：
        1.  **快捷选项**：25分钟、40分钟（可点击）
        2.  **自由输入**：提示用户直接发送数字，如“30”
    -   用户点击或输入后，系统最终确认并开始计时：“开始【阅读】，时长30分钟。放下手机，专注当下～”
-   **按钮状态变化**
    -   无进行中活动：底部菜单固定显示 **“开始活动”**。
    -   有进行中活动：底部菜单自动切换为 **“结束”**。

### 4.3 结束活动与番茄反馈

#### 情形一：用户主动点击菜单“结束”按钮（或发送“结束”）

| 当前状态 | 系统行为 | 获得反馈 |
| :--- | :--- | :--- |
| **提前结束**<br>(当前时间 < 计划结束时间) | **需要二次确认**：弹出提示“确定要提前结束【阅读】吗？你将获得一颗半熟番茄🍅”。提供“确定”和“继续做”按钮。 | 半熟番茄 |
| **超时完成**<br>(当前时间 ≥ 计划结束时间) | **无需确认**，直接结束并记录实际用时。 | 完美番茄 |

#### 情形二：用户通过“时间到”提醒消息里的“完成”按钮结束

-   **系统行为**：点击即视为主动完成，**无需二次确认**，直接结束并记录实际用时。
-   **获得反馈**：完美番茄。

### 4.4 时间到与超时提醒

-   **第一次提醒**：设定时长结束时，推送模板消息：“【阅读】时间到！点击「完成」收获完美番茄吧～”（消息内含“完成”按钮）。
-   **第二次提醒**：若活动开始后，用户**30分钟内**仍未进行任何结束操作，系统再次推送模板消息：“【阅读】已经超过30分钟没有新活动了哦，还在继续吗？点击菜单「结束」即可完成，或直接开始下一个活动。”
-   **超时完成规则**：只要用户在**当天24：00前**点击了“完成”或通过菜单“结束”，即使已超时，仍然记为**完美番茄**，并如实记录实际结束时间。

### 4.5 今日时间线查看

-   **入口**：底部菜单“今日记录”，或发送“今日记录”。
-   **回复格式**：
